# 教师 AI 助手 PRD

版本：v0.1  
日期：2026-07-02  
状态：草稿

## 1. 产品定位

教师 AI 助手是一款面向中小学教师的备课与作业批改工具。第一版聚焦两个高频场景：生成教案、批改单题作业。

第一版不做职称、升迁、复杂教务管理和多 Agent 协作。

## 2. 目标用户

核心用户：

- 中小学一线教师
- 需要频繁备课、写教案、批改作业的教师

早期优先用户：

- 语文、数学、英语教师
- 初中教师
- 有固定教材版本和作业批改需求的教师

## 3. 核心问题

教师高频耗时工作主要集中在：

- 教案撰写重复度高
- 不同年级、学科、课型需要反复调整教案
- 作业批改耗时，尤其是主观题、解答题、作文类反馈
- 批改后还需要总结共性问题和讲评建议

## 4. 产品目标

MVP 目标：

- 让教师在 3 分钟内生成一份可编辑教案
- 让教师在 1 分钟内完成一道主观题/解答题批改
- 输出内容必须可复制、可修改、可用于教学工作

非目标：

- 不替代教师最终判断
- 不直接给学生自动定最终成绩
- 不做学校级教务系统
- 不处理大规模学生隐私数据

## 5. MVP 功能范围

### 5.1 教案生成

用户输入：

- 学段：小学 / 初中 / 高中
- 年级
- 科目
- 教材版本
- 课题
- 课时
- 课型：新授课 / 复习课 / 讲评课 / 公开课
- 教学风格：常规 / 探究式 / 任务驱动 / 公开课
- 补充要求

系统输出：

- 教学目标
- 教学重难点
- 教学准备
- 教学过程
- 课堂提问
- 学生活动
- 板书设计
- 作业设计
- 教学反思模板

### 5.2 单题作业批改

用户输入：

- 题目
- 标准答案
- 评分规则
- 满分
- 学生答案
- 反馈风格：简洁 / 鼓励 / 严格 / 详细

系统输出：

- 建议得分
- 扣分点
- 错误原因
- 修改建议
- 给学生的反馈
- 给教师的讲评建议

## 6. 用户流程

### 6.1 教案生成流程

1. 用户进入“教案生成”
2. 选择年级、科目、教材版本
3. 输入课题和课时
4. 选择课型和教学风格
5. 点击生成
6. 系统返回结构化教案
7. 用户复制、修改或重新生成

### 6.2 作业批改流程

1. 用户进入“作业批改”
2. 输入题目、标准答案、评分规则、学生答案
3. 点击批改
4. 系统返回建议得分和反馈
5. 用户确认、修改或重新批改

## 7. 页面原型说明

第一版使用单页应用，包含两个 Tab：

- 教案生成
- 作业批改

页面区域：

- 左侧：输入表单
- 右侧：AI 输出结果
- 顶部：功能切换
- 底部：复制、重新生成、清空按钮

## 8. AI 能力要求

### 8.1 教案生成要求

- 输出必须结构化
- 内容要符合所选学段和年级
- 不得编造教材页码、课标原文
- 如果信息不足，应提示用户补充
- 公开课风格可以更详细，常规课风格应更实用

### 8.2 作业批改要求

- 必须基于评分规则批改
- 不应在没有评分规则时给出过度确定的分数
- 对主观题给出“建议得分”，最终分数由教师确认
- 反馈语言应适合学生阅读
- 对错误原因要具体，不只说“答案不完整”

## 9. 数据与隐私

第一版原则：

- 不要求用户登录
- 不保存学生姓名
- 不上传真实敏感身份信息
- 本地开发阶段不接入正式数据库

后续如加入历史记录，需要明确：

- 保存哪些数据
- 保存多久
- 用户如何删除
- 是否包含学生个人信息

## 10. 质量评测

### 10.1 教案质量指标

- 结构完整率
- 年级适配度
- 学科适配度
- 可直接修改使用程度
- 是否出现明显事实错误

### 10.2 批改质量指标

- 分数合理性
- 扣分点准确性
- 反馈可读性
- 是否严格依据评分规则
- 是否避免过度武断

### 10.3 测试样例

至少准备：

- 3 个教案生成样例
- 3 个数学解答题批改样例
- 3 个语文阅读题批改样例
- 3 个英语作文反馈样例

## 11. 技术方案

建议 MVP 技术栈：

- 前端：Streamlit 或 React
- 后端：Python
- LLM 调用：OpenAI 兼容接口
- 配置管理：.env
- 版本管理：Git

推荐第一版先使用 Streamlit，原因是开发快，适合验证需求。

后续产品化再切换为：

- 前端：React / Vue
- 后端：FastAPI
- 数据库：SQLite / PostgreSQL
- 文档检索：向量数据库 + RAG

## 14. 风险

- AI 输出可能不稳定，需要人工确认
- 批改主观题存在争议，必须标注“建议得分”
- 教材和课标内容如果没有资料支撑，不能假装引用
- 学生作业数据涉及隐私，后续必须做权限和脱敏
