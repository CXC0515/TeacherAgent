# Google AI Studio 原型提示词

用途：生成教师 AI 助手的中保真单文件 HTML 原型。  
使用方式：复制下方完整提示词到 Google AI Studio。

```text
你是一名资深 Apple Human Interface 风格的 Web SaaS 产品设计师、教育科技产品经理和前端原型工程师。请生成一个高级 Apple 风格、中保真、可交互的单文件 HTML 原型。

重要要求：
- 当前只做产品原型，不做真实后端。
- 使用单文件 HTML + CSS + 原生 JavaScript。
- 不使用 React、Vue、Tailwind、Bootstrap、CDN 或构建工具。
- 所有数据使用模拟数据。
- 输出完整 HTML 代码，包含 style 和 script。
- 原型重点是“层级关系”和“工作流交互”，不要把所有页面平铺成一堆卡片。
- 界面要像真实教师工作系统，不要营销页，不要大 hero，不要装饰性插画。
- 风格：高级 Apple 风格 + 教育 SaaS 后台 + 一点轻未来感。
- 代码结构清晰，方便未来迁移到 React + TypeScript + FastAPI。

产品名称：
教师 AI 助手

产品定位：
面向初中语文教师的教学证据采集、作业 AI 批改、学情诊断、学生画像和教学管理系统。

核心理念：
系统不是简单批改作业，而是把学生作业、知识点、题目、班级、学生表现、家校信息、日常观察等数据组织起来，辅助教师持续掌握班级和学生状态。

整体布局：
- 左侧为可展开的分组导航。
- 顶部为状态栏。
- 中间为主工作区。
- 右侧在部分页面显示详情抽屉。
- 支持 modal、toast、步骤工作流、可展开面板。

顶部状态栏：
显示：
- 当前学期：2026 春季学期
- 当前班级：七年级 3 班
- 当前学科：初中语文
- 教材：统编版七年级下册
- 当前教师：王老师
- 按钮：新建任务、进入班级

左侧导航层级：
1. 工作台
2. 管理
   - 班级管理
   - 学生管理
   - 课表与提醒
3. AI 批改
   - 作业批改工作流
   - 复核队列
4. 学情诊断
   - 班级诊断
   - 学生画像
5. 资料库
   - 知识图谱
   - 资料编辑
6. 设置

视觉风格要求：
整体采用高级 Apple 风格，但不要做成手机 App，要保持专业桌面 Web SaaS 感。

设计关键词：
- Apple-like
- 精致
- 克制
- 半透明
- 磨砂玻璃
- 柔和阴影
- 莫兰迪色系
- 高级教育 SaaS
- 轻未来感

具体视觉要求：
1. 使用 Apple 风格圆角：
   - 大容器圆角 24px
   - 卡片圆角 18px
   - 按钮圆角 12px
   - 标签圆角 999px

2. 使用半透明磨砂玻璃效果：
   - sidebar、topbar、modal、floating panel、student detail drawer 使用 backdrop-filter: blur(...)
   - 背景使用柔和的莫兰迪渐变或低饱和色块
   - 不要使用强烈霓虹色，不要赛博朋克过度发光

3. 阴影要柔和：
   - 使用多层 box-shadow
   - 避免硬边阴影
   - 卡片悬停时轻微上浮

4. 默认色系为莫兰迪色系：
   - 雾蓝
   - 灰绿
   - 豆沙粉
   - 暖灰
   - 米白
   - 烟紫
   - 石墨灰
   但整体不要花，主界面以浅灰白、雾蓝、灰绿为主。

5. 保留主题色选择切换功能：
   设置页中提供主题选择：
   - 莫兰迪绿，默认
   - 雾蓝
   - 豆沙粉
   - 暖灰
   - 深色石墨
   点击后通过 CSS 变量切换主题色。

6. 支持深色模式自动适配：
   使用 prefers-color-scheme: dark。
   深色模式下：
   - 背景变为深石墨色
   - 卡片为半透明深灰
   - 字体为柔和白
   - 边框为低透明白
   - 主题色仍保持低饱和
   - 图谱节点和状态灯保持可辨识

7. 动效要求：
   - 所有 hover、active、切页、modal、toast、drawer 展开都要有柔和过渡
   - 使用 transform 和 opacity 实现动效，避免频繁触发布局重排
   - transition 时间控制在 160ms-260ms
   - 工作流节点切换要有轻微滑入或淡入
   - 虚拟教室点击学生时，右侧详情面板平滑滑出
   - 卡片 hover 轻微 translateY(-2px)
   - 按钮点击有 scale(0.98) 触感反馈

8. 性能要求：
   - 不要使用大量复杂动画
   - 不要使用 canvas
   - 不要使用外部库
   - 动效只使用 CSS transition / transform / opacity
   - JS 只负责切换 class 和状态
   - 代码要保持轻量

9. 组件质感：
   - 表格要像 Apple 系统设置或高级后台，不要粗糙边框
   - 输入框使用半透明背景和细边框
   - 状态标签使用低饱和色
   - 复核队列、工作流、知识图谱、虚拟教室都要统一视觉语言
   - modal 和 drawer 要有磨砂玻璃、柔和阴影和背景遮罩

10. 可访问性：
   - 深浅模式下文字对比度要足够
   - 状态不能只靠颜色，也要有文字标签
   - 重要按钮要明显但不刺眼

页面一：工作台
要求：
- 显示待处理任务，而不是单纯统计卡片。
- 任务按节点分类：
  - 待上传作业
  - 待识别校对
  - 待 AI 评分
  - 待人工复核
  - 待生成讲评
  - 今日提醒
- 每个任务显示：
  - 任务名称
  - 所属班级
  - 当前节点
  - 截止时间
  - 状态
  - 操作按钮
- 有一个醒目的“进入班级”入口。

工作台还要显示：
- 今日课表
- 收作业提醒
- 最近班级风险
- 最近学生异常表现
- 最近 AI 批改任务进度

页面二：进入班级 / 虚拟教室视图
这是特色页面。
要求：
- 做一个卡通 2D 或固定视角 3D 感的教室。
- 讲台在画面下方，学生座位在上方排布，模拟教师视角。
- 每个学生用简洁卡通头像或圆形人物表示，坐在课桌后。
- 每张课桌上有状态灯：
  - 绿色：状态良好
  - 黄色：需要关注
  - 红色：近期风险
  - 蓝色：表现突出
- 点击学生，右侧弹出动态面板。

学生动态面板显示：
- 姓名
- 所属班级
- 是否课代表
- 最近作业状态
- 日常表现标签
- 家庭关注信息摘要
- 学习状态灯
- 学得好的知识点/题型
- 学得不好的知识点/题型
- 最近三次作业趋势
- 按钮：查看完整画像、添加观察记录、设置提醒

页面三：管理 / 班级管理
要求：
- 表格 + 右侧详情抽屉。
- 班级字段：
  - 班级名称
  - 年级
  - 学期
  - 班主任
  - 任课教师
  - 教材版本
  - 学生人数
  - 课代表，可多选
  - 默认收作业时间
  - 状态
- 支持：
  - 新建班级
  - 编辑班级
  - 删除/归档班级
  - 设置课代表
  - 设置收作业提醒
  - 进入班级视图

页面四：管理 / 学生管理
要求：
- 支持按班级筛选。
- 学生表格字段：
  - 姓名
  - 学号
  - 所属班级
  - 性别
  - 是否课代表
  - 最近学习状态
  - 日常表现
  - 家长联系人
  - 家庭状态标签
  - 操作
- 支持：
  - 单独添加学生
  - 批量导入学生
  - 编辑学生
  - 删除/归档学生
  - 批量移动班级
  - 批量设置标签

学生详情中包含：
- 基础信息
- 所属班级
- 家长信息
  - 父母/监护人姓名
  - 联系方式
  - 关系
  - 沟通备注
- 家庭状态
  - 普通
  - 需关注
  - 留守/单亲/特殊情况等标签，使用模拟数据即可
- 日常表现
  - 课堂参与
  - 作业习惯
  - 情绪状态
  - 纪律表现
  - 教师观察记录
- 学习相关信息
  - 优势知识点
  - 薄弱知识点
  - 最近作业趋势

页面五：管理 / 课表与提醒
要求：
- 显示周课表。
- 可以选择班级。
- 支持手动添加：
  - 语文课
  - 会议
  - 教研
  - 收作业提醒
  - 家校沟通
  - 批改任务
- 课表格子点击后弹出编辑 modal。
- 支持定时提醒：
  - 每周一 8:00 收周末作业
  - 每周三 16:00 阅读训练收取
  - 考前复习提醒
- 提醒列表显示：
  - 提醒名称
  - 关联班级
  - 时间
  - 重复规则
  - 状态

页面六：AI 批改 / 作业批改工作流
这是核心页面。
要求：
- 不要做成普通表单。
- 做成一个可点击的步骤工作流。
- 顶部显示工作流节点：
  1. 设置作业任务
  2. 设置题目
  3. 上传标准答案与评分细则
  4. 上传学生作业
  5. 识别校对
  6. AI 评分
  7. 置信度复核
  8. 人工复核
  9. 生成作业诊断
  10. 同步到班级和学生画像
- 每个节点可以点击进入编辑。
- 已完成节点显示绿色。
- 当前节点高亮。
- 有问题节点显示黄色或红色。
- 支持回到之前节点修改、补充资料。

节点详情：
1. 设置作业任务：
   - 作业名称
   - 班级
   - 学科
   - 截止时间
   - 关联课文
   - 作业类型
2. 设置题目：
   - 选择题库题目
   - 手动添加题目
   - 从图片识别题目
3. 上传标准答案与评分细则：
   - 支持上传图片/文档
   - 支持结构化编辑评分点
4. 上传学生作业：
   - 拍照上传
   - 扫描上传
   - 批量上传
   - 显示上传进度
5. 识别校对：
   - 左侧作业图
   - 中间 OCR 文本
   - 右侧置信度和题目匹配
6. AI 评分：
   - 显示建议分
   - 评分点命中
   - 扣分点
   - 错误类型
7. 置信度复核：
   - 低于设置阈值的进入复核
   - 阈值默认 0.75
8. 人工复核：
   - 分差大的送人工复核
   - 多模型或多教师结果不一致也送复核
9. 生成作业诊断：
   - 班级共性问题
   - 题目得分情况
   - 高频错因
10. 同步画像：
   - 将“某学生在某次作业某题某知识点错误”写入学生画像和班级诊断

页面七：AI 批改 / 复核队列
要求：
- 分为：
  - 低置信度
  - 分差过大
  - AI 冲突
  - 待主批确认
  - 已完成
- 列表按优先级排序。
- 点击一条后显示：
  - 学生答案
  - 标准答案
  - 评分细则
  - AI 建议分
  - 教师最终分
  - 差异原因
  - 证据文本
- 操作：
  - 确认
  - 修改分数
  - 退回识别校对
  - 标记为样例
  - 加入讲评材料

页面八：学情诊断 / 班级诊断
要求：
- 可以选择班级、时间范围、作业任务。
- 显示：
  - 班级整体趋势
  - 本次作业表现
  - 知识点掌握热力图
  - 题型薄弱排行
  - 高频错误类型
  - 需要重点关注学生
  - 可生成讲评建议
- 点击某个知识点，可以看到：
  - 哪些题考了这个知识点
  - 哪些学生错了
  - 错在什么类型
  - 最近是否反复出现

页面九：学情诊断 / 学生画像
要求：
- 左侧学生列表。
- 右侧学生画像。
- 画像分区：
  1. 基础信息
  2. 家长与家庭信息
  3. 日常表现
  4. 学习优势
  5. 学习薄弱点
  6. 作业记录
  7. 知识点错误记录
  8. 教师观察
- 知识点错误记录示例：
  - 时间：2026-03-12
  - 作业：《驿路梨花》课后阅读练习
  - 题目：标题作用题
  - 知识点/能力点：结构分析、主旨理解
  - 错误类型：漏答要点、没有结合文本
  - 当前状态：待巩固
- 学得好的和学得不好的要分开展示。

页面十：资料库 / 知识图谱
默认是“高大上的赛博知识图谱视图”。
要求：
- 用 CSS 模拟节点和连接线。
- 节点类型：
  - 课本
  - 单元
  - 课文
  - 题目
  - 知识点
  - 能力点
  - 错误类型
- 节点有不同颜色。
- 点击节点打开详情面板。
- 有切换按钮：
  - 图谱视图
  - 编辑视图

图谱示例：
统编版七下 -> 第四单元 -> 《驿路梨花》 -> 标题作用题 -> 结构分析 / 主旨理解 -> 漏答要点

页面十一：资料库 / 编辑视图
相当于编辑数据库。
要求：
- 左侧对象类型：
  - 课本
  - 单元
  - 课文
  - 题目
  - 知识点
  - 能力点
  - 错误类型
- 中间表格显示对象列表。
- 右侧详情编辑：
  - 名称
  - 类型
  - 描述
  - 关联对象
  - 前置关系
  - 后续关系
  - 权重
- 支持：
  - 选择预设课本
  - 添加题目
  - 添加知识点
  - 编辑知识点之间的拓扑关系
  - 删除关系
  - 保存

页面十二：设置
包含：
- 低置信度阈值，默认 0.75
- 分差复核阈值，默认 1 分
- 是否默认隐藏高置信度结果
- 是否开启学生姓名脱敏
- 原图保存策略
- 提醒设置
- 默认班级
- 默认教材
- 主题选择：
  - 莫兰迪绿，默认
  - 雾蓝
  - 豆沙粉
  - 暖灰
  - 深色石墨

交互要求：
1. 左侧导航支持展开和折叠。
2. 点击二级菜单切换对应页面。
3. 工作台任务点击后跳转到对应模块。
4. 点击“进入班级”切换到虚拟教室页面。
5. 虚拟教室点击学生，弹出学生动态面板。
6. 班级管理可打开编辑 modal。
7. 学生管理可添加、编辑、删除、批量导入模拟。
8. 课表格子点击可编辑。
9. AI 批改工作流节点可点击切换，节点状态可变化。
10. 作业上传按钮点击后模拟进度变化。
11. 识别校对确认后显示 toast。
12. 复核队列确认后状态变为已完成。
13. 知识图谱支持图谱视图和编辑视图切换。
14. 设置里调整阈值后，复核队列显示的阈值同步变化。
15. 设置里切换主题后，通过 CSS 变量更新主题色。

视觉补充：
- 不要所有内容都用大卡片堆起来。
- 多用表格、列表、详情抽屉、步骤流、状态标签、筛选器。
- 虚拟教室可以更有视觉特色，但不要幼稚。
- 知识图谱要有赛博感，但整体仍要专业。
- 所有按钮、状态、字段都要像真实产品。
- 页面文案要简洁，不要解释产品理念。

输出要求：
- 输出完整 HTML 文件代码。
- CSS 写在 style 标签中。
- JS 写在 script 标签中。
- 不依赖外部资源。
- 不要使用真实图片，用 CSS 模拟。
- 不要输出解释，只输出代码。
```
