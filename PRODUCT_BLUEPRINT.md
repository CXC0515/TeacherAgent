# 教师 AI 助手产品总蓝图

版本：v0.2  
日期：2026-07-03  
状态：待评审

## 1. 核心判断

本产品不是单点的教案生成器，也不是简单的作业批改工具。

它的长期形态应是：

```text
围绕教师、学生、教材、作业、知识点和教学证据构建的 AI 教学工作系统。
```

核心闭环：

```text
教学计划 -> 课堂教学 -> 作业采集 -> 作答识别 -> 评价分析 -> 教学干预 -> 再教学
```

第一性原理：

```text
教师真正需要的不是 AI 生成文本，而是把教学过程中的真实证据转化为可行动的判断。
```

## 2. 长期产品定位

产品定位：

```text
面向教师的教学证据采集、分析与行动建议系统。
```

核心价值：

- 减少教师重复性事务
- 沉淀学生学习过程数据
- 建立知识点、题目、学生表现之间的关系
- 帮助教师发现班级和个体的真实问题
- 生成讲评、补救、备课、沟通等行动建议

第一阶段产品切入：

```text
初中语文任课教师的作业图片采集、AI 初评、教师复核、班级诊断和学生长期画像。
```

## 3. 系统分区

产品应分为八个区域，而不是一开始只做单个页面。

### 3.1 教师工作区

管理教师当前教学身份和工作范围。

包含：

- 当前学期
- 任教学段
- 年级
- 科目
- 教材版本
- 任教班级
- 是否班主任
- 教学时间范围
- 教学进度

### 3.2 班级与学生区

管理班级和学生基础信息。

包含：

- 班级
- 学生
- 分组
- 学生标签
- 学生长期画像
- 学生阶段性变化
- 学生名单手动录入
- Excel 导入
- 学生字段扩展
- 学生删除/归档

### 3.3 教材与课程区

管理教材、章节、课时和教学进度。

包含：

- 教材版本
- 册别
- 单元
- 课文 / 课题
- 课时
- 教学目标
- 单元目标
- 课程标准关联
- 教材/课文全文存储
- 内容索引
- 关键词触发式检索
- 全文检索和语义检索

### 3.4 知识与能力图谱区

管理知识点、能力点、题型、评价维度之间的关系。

包含：

- 知识点
- 能力点
- 题型
- 错误类型
- 评分维度
- 前置关系
- 承接关系
- 包含关系
- 迁移关系

对于语文，不能只用“知识点”描述，应同时包含：

- 文体
- 题型
- 阅读能力
- 表达能力
- 思维能力
- 评分点
- 常见失分类型

### 3.5 作业与测评区

管理作业、题目、评分标准和学生提交。

包含：

- 作业
- 题目
- 电子题目
- 阅读材料
- 参考答案
- 评分规则
- 题目与知识点/能力点关联
- 学生提交
- 作业图片
- 作答区域
- OCR 识别文本
- 多模态识别结果
- 教师校对记录

### 3.6 AI 识别与批改区

处理从原始作业到结构化评价结果的过程。

包含：

- 图片上传
- 整页识别
- 页面边缘识别
- OCR 识别
- 版面切分
- 学生答案定位
- 同一学生多页合并
- 答案文本清洗
- AI 初评
- 评分点匹配
- 错误类型判断
- 教师确认
- 低置信度复核队列
- 多人/多模型冲突复核
- 版本记录

原则：

```text
AI 只给建议，教师拥有最终确认权。
```

### 3.7 分析诊断区

把作业结果转化为教学判断。

包含：

- 班级整体表现
- 题目得分分布
- 知识点掌握情况
- 能力点薄弱情况
- 共性错误
- 个体学生问题
- 学生变化趋势
- 学生长期画像
- 班级分层
- 高风险学生识别

### 3.8 教学行动区

把分析结果转化为教师下一步动作。

包含：

- 讲评课方案
- 二次训练题
- 个别辅导建议
- 分层作业建议
- 家校沟通建议
- 下一课备课调整
- 教学反思
- 教研材料沉淀

## 5. 字段设计原则

字段不能只记录值，还要记录来源和可信度。

通用字段属性：

```text
value          字段值
type           数据类型
source         来源：人工录入 / OCR / AI 推断 / 系统生成
confidence     置信度
created_at     创建时间
updated_at     更新时间
version        版本
privacy_level  隐私等级
verified_by    确认人
verified_at    确认时间
```

示例：

```text
RecognizedText
- text
- source: OCR / multimodal_ai / teacher_corrected
- confidence
- linked_image_region
- verified_by_teacher
```

## 6. 关系设计原则

关系也应该是对象，不能只靠外键。

关系字段：

```text
from_object
to_object
relation_type
weight
confidence
evidence
valid_from
valid_to
created_by
```

典型关系：

```text
Teacher -> teaches -> Class
Class -> contains -> Student
Textbook -> contains -> Unit
Unit -> contains -> Lesson
Lesson -> involves -> KnowledgePoint
KnowledgePoint -> prerequisite_of -> KnowledgePoint
Question -> assesses -> KnowledgePoint
Question -> assesses -> AbilityPoint
Student -> submitted -> Submission
Submission -> contains -> RecognizedText
GradingResult -> based_on -> Evidence
Student -> weak_at -> AbilityPoint
Diagnosis -> suggests -> Intervention
```

## 7. 多维分析模型

系统至少应支持这些维度：

```text
时间维度：学期、周次、课时、作业时间、变化趋势
人员维度：教师、班级、学生、分组
教材维度：教材版本、单元、课文、课时
知识维度：知识点、能力点、题型、评分点、错误类型
证据维度：图片、OCR 文本、AI 判断、教师确认
行动维度：讲评、补救、分层训练、家校沟通、备课调整
```

## 8. AI 的位置

AI 不是系统本体，而是多个环节中的能力层。

AI 主要用于：

- 从图片识别学生答案
- 识别整页边缘、版面和作答区域
- 将题目图片或电子文档转为电子题目
- 把题目映射到知识点/能力点
- 根据评分标准进行初评
- 按置信度生成复核队列
- 总结共性错误
- 生成讲评建议
- 生成补救练习
- 生成教学反思
- 解释学生长期变化

必须由教师确认的内容：

- 学生最终得分
- 关键批改结论
- 重要学生画像
- 对学生和家长输出的正式反馈
- 低置信度和冲突结果

## 9. 分阶段实现路线

### 9.1 总体原则

先扩清楚全局模型，再分区域实现。

实现顺序：

```text
数据对象 -> 关系模型 -> 采集流程 -> AI 处理 -> 分析诊断 -> 教学行动
```

### 9.2 第一阶段：系统骨架

目标：

- 建立教师、班级、学生、教材、课程、知识/能力点、作业、题目的基础对象
- 支持学生手动录入、Excel 导入、编辑、删除和字段扩展
- 支持初中语文的文体、题型、能力点、评分点和错误类型
- 建立对象之间的关系
- 为长期学生画像预留数据结构

### 9.3 第二阶段：作业证据采集

目标：

- 支持移动端拍照和网页端上传
- 支持整页作业图片上传
- 支持边缘识别、裁剪、同一学生多页合并和题目匹配
- 支持 OCR / 多模态识别
- 支持教师校对识别结果
- 保存原图、识别文本、校对文本之间的证据链

### 9.3.1 题目电子化

目标：

- 支持题目图片识别
- 支持电子版本导入
- 支持识别阅读材料、题干、参考答案和评分标准
- 形成可复用的电子题目

### 9.4 第三阶段：AI 初评与教师确认

目标：

- 根据题目、参考答案、评分规则进行 AI 初评
- 输出建议得分、扣分点、错误类型
- 教师确认或修改
- 按置信度从低到高排序复核
- 高置信度结果可默认折叠
- 多人或多模型不一致时进入冲突复核

### 9.5 第四阶段：诊断分析

目标：

- 形成班级、学生、知识点、能力点的多维分析
- 输出共性问题和个体问题
- 更新学生长期画像

### 9.6 第五阶段：教学行动生成

目标：

- 生成讲评课
- 生成补救练习
- 生成分层作业
- 生成个别辅导建议

### 9.7 第六阶段：长期画像

目标：

- 追踪学生能力变化
- 追踪班级薄弱点变化
- 支持跨单元、跨学期分析
- 目标上支持初中三年连续学习过程

## 10. 初中语文扩展示例

初中语文不是简单的对错判断，应以真实文本表达为核心证据。

语文维度：

```text
文体：记叙文 / 说明文 / 议论文 / 文言文 / 古诗文
题型：概括题 / 赏析题 / 作用题 / 含义题 / 人物形象题 / 主旨题
能力点：信息提取 / 文意理解 / 语言赏析 / 结构分析 / 表达组织
评分点：关键词 / 答题角度 / 文本依据 / 表达完整性
错误类型：漏点 / 偏题 / 空泛 / 没结合文本 / 术语误用 / 表达混乱
```

示例关系：

```text
阅读题 Q1 -> assesses -> 信息提取
阅读题 Q1 -> has_rubric_point -> 关键词完整
学生答案 A1 -> has_error_type -> 漏点
学生答案 A1 -> evidence_of -> 信息提取能力不足
班级诊断 D1 -> suggests -> 讲评“如何找准答题区间”
```

## 11. 本轮评审结论

已确认的产品方向：

- 第一阶段以初中语文为主，其他学科保留扩展模型。
- 作业采集优先支持整页上传，再做边缘识别、裁剪、学生多页合并和题目识别。
- 学生名单同时支持手动录入和 Excel 导入，并支持编辑、删除和字段扩展。
- 题目和评分标准支持图片识别、电子版本导入，并沉淀为电子题目。
- 教材和课文内容允许存储，检索层支持关键词触发、全文检索和语义检索。
- 教师复核采用置信度排序，高置信度可折叠，低置信度和冲突结果进入复核机制。
- 学生画像从第一阶段开始写入长期模型，目标上支持初中三年连续追踪。
- 家校沟通先不做核心功能，但保留班主任权限和沟通对象扩展。
- 第一阶段先做任课教师工作流。
- 移动端拍照和网页端上传都需要纳入设计。

## 12. 产品边界

短期不做：

- 全学科一次性覆盖
- 学校级教务管理
- 自动替代教师最终评分
- 高风险学生结论自动下发
- 无证据的学生能力判断

长期可拓展：

- 教案生成
- 作文诊断
- 口语/朗读分析
- 班主任事务
- 家校沟通
- 教研材料
- 职称材料
